# BAY 생성 정책

- BAY 템플릿은 새 BAY 작업 구성을 시작하기 위한 초기 바탕이다.
- 사용자는 템플릿에서 복사된 작업 그룹과 상세 작업을 자유롭게 추가, 삭제, 수정, 정렬할 수 있다.
- 생성된 BAY의 작업 목록은 원본 템플릿과 완전히 독립적이다.
- 템플릿을 나중에 변경해도 이미 생성된 BAY에는 영향을 주지 않는다.
- BAY에는 템플릿 ID나 템플릿 버전을 저장하지 않는다.
- 생성 API는 템플릿 참조가 아니라 사용자가 최종 확정한 작업 목록 전체를 입력받는다.
- BAY 레코드와 작업 목록은 하나의 데이터베이스 트랜잭션에서 저장한다.
- BAY 코드 형식, 중복 여부, 작업 목록의 구조와 순서는 서버에서 다시 검증한다.
- 신규 작업의 진행 상태, 이슈 상태, 작업자와 작업일은 초기값으로 저장한다.
